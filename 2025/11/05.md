## わかったこと

### RESTの主な原則（6つの特徴）
| 原則                                  | 内容                              | 具体例                      |
| :---------------------------------- | :------------------------------ | :----------------------- |
| **① クライアント／サーバー構造**                 | クライアント（ブラウザなど）とサーバー（API提供側）を分ける | WebブラウザがAPIサーバーにリクエストを送る |
| **② ステートレス**                        | 各リクエストは独立しており、サーバーは状態を保持しない     | 毎回ログイン情報や必要データを送る        |
| **③ キャッシュ可能**                       | レスポンスをキャッシュ（再利用）できる             | 同じデータを繰り返し取得しないようにする     |
| **④ 統一インターフェース（Uniform Interface）** | 一貫したルールで操作できる                   | URLとHTTPメソッドの組み合わせで操作を表す |
| **⑤ 階層化システム**                       | 中継サーバーやロードバランサーなどを間に挟んでもOK      | APIゲートウェイなど              |
| **⑥ コードオンデマンド（任意）**                 | 必要に応じてスクリプトを配信できる               | 例：JavaScriptを返すことも可能（任意） |

### RESTに沿ったWeb APIの例
| 操作    | HTTPメソッド        | URL（エンドポイント）   | 説明           |
| :---- | :-------------- | :------------- | :----------- |
| 一覧を取得 | **GET**         | `/employees`   | すべての社員データを取得 |
| 1件取得  | **GET**         | `/employees/1` | ID=1の社員を取得   |
| 新規登録  | **POST**        | `/employees`   | 新しい社員を追加     |
| 更新    | **PUT / PATCH** | `/employees/1` | ID=1の社員を更新   |
| 削除    | **DELETE**      | `/employees/1` | ID=1の社員を削除   |
